## Concepts

- **Tool Overview**: Converts 10x Genomics BAM files (from Cell Ranger, Space Ranger, Cell Ranger ATAC, Cell Ranger DNA, Long Ranger) back to FASTQ format for re-analysis. Version 1.4.1.
- **Core Function**: Extracts reads from 10x BAM files and writes them to FASTQ files while preserving barcode information in read headers.
- **Input/Output**: Input is a 10x BAM file; output is FASTQ files in a specified directory. Output FASTQs contain the same sequences as input but order is not preserved.
- **Installation**: Install via bioconda: `conda install -c bioconda 10x_bamtofastq`
- **Platform Support**: Linux (x86_64, ARM64) and macOS (Intel, Apple Silicon)
- **Threading**: Supports multi-threaded BAM reading via `--nthreads` option. Default is 4 threads.
- **Version Compatibility**: Has flags for different 10x pipeline versions (--gemcode, --lr20, --cr11).

## Pitfalls

- **Version Differences**: Different 10x pipeline versions require different flags (--gemcode for Longranger 1.0-1.3, --lr20 for Longranger 2.0, --cr11 for Cell Ranger 1.0-1.1). Using wrong flag may produce incorrect output.
- **V(D)J Data Unsupported**: BAM files from V(D)J analysis (TCR/BCR) cannot be converted with this tool.
- **Long Runtime**: Full-coverage WGS BAM files may take hours to process. Plan accordingly.
- **Order Not Preserved**: Output FASTQ sequence order differs from input BAM. Downstream tools must not assume order consistency.
- **BX Barcode Preservation**: Barcode information is preserved in read headers. Ensure downstream tools can parse 10x-formatted FASTQ headers.

## Examples

### Display help and version information
**Args:** `--help`
**Explanation:** Shows all available command-line options, flags, and usage information. Essential for discovering available parameters.

### Basic conversion with multi-threading
**Args:** `--nthreads=8 input.bam output_dir/`
**Explanation:** Converts a 10x BAM file to FASTQ using 8 threads for faster processing. Output FASTQ files are written to the specified directory.

### Convert Cell Ranger 1.0-1.1 BAM file
**Args:** `--cr11 input.bam output_dir/`
**Explanation:** Converts BAM files generated by older Cell Ranger versions (1.0-1.1). The --cr11 flag ensures correct parsing of the legacy BAM format.

### Convert Longranger 2.0 BAM file
**Args:** `--lr20 input.bam output_dir/`
**Explanation:** Converts BAM files from Longranger 2.0 pipeline. Required for proper handling of Longranger 2.0 specific format.

### Filter reads by genomic locus
**Args:** `--locus=chr1:1000-2000 --nthreads=4 input.bam output_dir/`
**Explanation:** Extracts only reads mapping to the specified genomic region (chr1:1000-2000). Useful for targeted re-analysis of specific regions.

### Convert GemCode data (Longranger 1.0-1.3)
**Args:** `--gemcode input.bam output_dir/`
**Explanation:** Converts BAM files from GemCode/Longranger 1.0-1.3 era. Required for legacy data processing.

### Adjust FASTQ chunk size
**Args:** `--reads-per-fastq=10000000 --nthreads=8 input.bam output_dir/`
**Explanation:** Sets each output FASTQ file to contain 10 million reads (default is 50 million). Smaller chunks are easier to handle in downstream pipelines.

### Filter by barcode list
**Args:** `--bx-list=barcodes.txt input.bam output_dir/`
**Explanation:** Extracts only reads with BX barcodes listed in the text file. Requires BX-sorted and indexed BAM file. Useful for isolating specific cells or samples.
