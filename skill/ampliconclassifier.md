## Concepts

- **Tool Overview**: AmpliconClassifier classifies output from AmpliconArchitect to detect different types of focal amplifications in cancer genomes, including extrachromosomal DNA (ecDNA) and breakage-fusion-bridge (BFB) cycles. Version 0.4.14.
- **Amplification Types**: Identifies ecDNA, BFB cycles, complex non-cyclic amplifications, and other focal amplification structures from whole-genome sequencing data.
- **Integration with AmpliconArchitect**: Designed to process AA output files (cycles and graph files) generated from CNV and SV analysis.
- **Reference Genomes**: Supports hg19, GRCh37, GRCh38, mm10, and GRCm38. Requires AA data repository with reference genome annotations.
- **Classification Algorithm**: Uses decomposition analysis and edge classification profiles to determine amplification type and structure.
- **Visualization**: Optional radar-style classification intensity plots showing confidence in each amplification type.
- **Batch Processing**: Can classify multiple amplicons from multiple samples in a single run using `--AA_results` directory.

## Pitfalls

- **Environment Setup**: Requires setting `AC_SRC` environment variable to source directory and `AA_DATA_REPO` to reference genome data.
- **Input Requirements**: Needs both cycles file and graph file from AmpliconArchitect output for proper classification.
- **Chromosome Naming**: Use `--add_chr_tag` when mixing hg19 and GRCh37 amplicons to standardize chromosome naming.
- **Low-Complexity Filtering**: Default filters low-complexity and low-mappability regions. Disable with `--no_LC_filter` for specialized analyses.
- **Minimum Size Threshold**: Default min_size=5000bp may miss small focal amplifications. Adjust for targeted panels.
- **Reference Data**: Must download AA data repository for chosen reference genome before running classification.

## Examples

### Classify multiple amplicons from AA results
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/output/ > classifier.log`
**Explanation:** Standard workflow: classifies all amplicons from AmpliconArchitect output directory. Outputs classification results to stdout and log file.

### Classify single amplicon
**Args:** `python amplicon_classifier.py --ref GRCh38 --cycles sample_cycles.txt --graph sample_graph.txt -o sample_classified`
**Explanation:** Classifies a single amplicon using specific cycles and graph files. Output prefix specified with -o.

### Use pre-made input file
**Args:** `python amplicon_classifier.py --ref GRCh38 --input collection.input -o results/`
**Explanation:** Uses input file generated by make_input.sh script. Useful for batch processing with custom sample groupings.

### Generate detailed classification profiles
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/ --verbose_classification -o detailed/`
**Explanation:** Enables verbose output including edge_classification_profiles.tsv. Useful for understanding classification decisions and debugging.

### Create results summary table
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/ --make_results_table -o summary/`
**Explanation:** Generates _results_table.tsv summarizing all classifications. Recommended for multi-sample analyses.

### Adjust decomposition strictness
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/ --decomposition_strictness 0.2 --min_flow 2`
**Explanation:** Increases decomposition strictness to 0.2 (default 0.1) and minimum cycle CN flow to 2. More stringent filtering for complex amplicons.

### Filter similar amplicons (independent samples)
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/ --filter_similar`
**Explanation:** Applies similarity-based filtering to remove potential false positive amplicons. Use only for samples from independent sources (not replicates).

### Generate classification intensity plots
**Args:** `python amplicon_classifier.py --ref GRCh38 --AA_results /path/to/AA/ --plotstyle individual -o plots/`
**Explanation:** Creates radar-style plots showing classification confidence for each amplification type. Experimental feature for visualization.
